# Agents Control Center - Basic Troubleshooting

## Command Not Found

Symptoms:

- `node is not recognized`
- `npm is not recognized`
- `openclaw is not recognized`
- `n8n is not recognized`

Fix:

1. Check whether the tool is installed.
2. Press `Add PATH` for that tool in the Install tab.
3. Close and reopen the app.
4. If Windows still does not recognize the command, restart Windows.

## npm Install Failed

Symptoms:

- npm permission error.
- package download failed.
- install command exits with error.

Fix:

1. Check internet access.
2. Check Node.js and npm versions.
3. Avoid mixing Microsoft Store Node with official Node.js.
4. Reinstall Node.js LTS if npm global path is broken.
5. Run the app with appropriate Windows permissions if the machine policy requires it.

## Port Already In Use

Symptoms:

- `EADDRINUSE`
- address already in use
- n8n or gateway cannot start.

Fix:

1. Stop the app from the Run tab.
2. Check whether another process is using the port.
3. For n8n, check port `5678`.
4. For OpenClaw Gateway, check port `18789`.
5. Change the configured port if needed.

## OpenClaw Gateway Does Not Start

Fix:

1. Confirm OpenClaw is installed.
2. Open Setup and review gateway configuration.
3. Run `Fix Gateway`.
4. Run `Doctor` if needed.
5. Restart Gateway.
6. Check logs if the problem continues.

## Dashboard Does Not Open

Fix:

1. Start or restart OpenClaw Gateway.
2. Press `Open Dashboard`.
3. If token auth is enabled, confirm the gateway token exists in `openclaw.json`.
4. Press Refresh if the status display is stale.

## Telegram Bot Does Not Respond

Check:

- `channels.telegram.enabled` is `true`.
- `channels.telegram.botToken` is set.
- `channels.telegram.defaultTo` points to the right chat/group.
- `channels.telegram.dmPolicy` is lowercase: `open`, `allowlist`, or `pairing`.
- `channels.telegram.groups["*"].requireMention` is correct for the group.
- `plugins.entries.telegram.enabled` is `true`.

If `tokenFile` points to a missing file while `botToken` exists, remove `tokenFile` or save the Telegram channel again in the app.

## ngrok Authentication Or Domain Error

Fix:

1. Enter a valid ngrok authtoken.
2. Enter the reserved domain without `https://`.
3. Remove trailing `/`.
4. Confirm the port is correct.
5. Confirm the account has permission to use the domain.

## API 401 Or 403

Fix:

1. Re-enter the API key.
2. Confirm the provider is correct.
3. Confirm the key has model access.
4. Save settings again.
5. Retry the action.

## Backup Or Restore Fails

Fix:

1. Use a valid backup zip generated by this app.
2. Do not edit the backup manifest manually.
3. Make sure the selected app matches the backup app.
4. Close running services before restore if files are locked.
